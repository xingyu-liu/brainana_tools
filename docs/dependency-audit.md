# Software & Dependency Audit — `brainana_tools`

> Audit of every third-party piece of software, library, runtime, and bundled
> asset used by the `brainana_tools` monorepo. Generated by static inspection of
> the manifests (`package.json`, `package-lock.json`), build/CI configuration,
> and the installed `node_modules` tree.
>
> **Audit date:** 2026-07-17 · **Repo version:** `brainana-viewer` 0.1.0

---

## 1. Scope & method

`brainana_tools` is a **Node.js / TypeScript npm workspace monorepo** — there is
no Python, Docker, or other-language toolchain in the repo. The audit covered:

| Source inspected | What it tells us |
| --- | --- |
| Root `package.json` | app runtime deps, dev/build deps, engines, browserslist |
| `apps/viewer/package.json` | domain-app deps |
| `packages/*/package.json` (7 packages) | shared-platform deps & peer deps |
| `package-lock.json` + `node_modules/*/package.json` | **actually installed** versions & licenses |
| `.github/workflows/ci.yml` | CI toolchain (GitHub Actions) |
| `apps/viewer/vite.config.ts`, `electron-builder.yml` | build/packaging tools |
| `packages/ui/fonts/`, `theme.css` | bundled/self-hosted assets |
| source `grep` for `node:*` | Node.js built-in modules relied on |

The repo declares only **4 distinct third-party runtime libraries** plus a small
set of build-time tools; everything else in `node_modules` is transitive.

## 2. Runtime dependencies (shipped / bundled)

These are the third-party libraries that end up in the running product — either
bundled into the browser SPA by Vite, or shipped as `node_modules` in the
packaged desktop app.

| Package | Declared range | Installed | License | Role in the project |
| --- | --- | --- | --- | --- |
| **[@niivue/niivue](https://github.com/niivue/niivue)** | `^0.69.0` | 0.69.0 | BSD-2-Clause | WebGL2 neuroimaging viewer — the core rendering engine for the viewer app. Consumed as a shared peer via `@brainana/niivue-kit`. |
| **[fflate](https://github.com/101arrowz/fflate)** | `^0.8.2` | 0.8.3 | MIT | Fast zlib/gzip (de)compression — used by `@brainana/core-client` and the viewer for compressed NIfTI/GIFTI payloads. |
| **[nifti-reader-js](https://github.com/rii-mango/NIFTI-Reader-JS)** | `^0.8.0` | 0.8.0 | MIT | Parses NIfTI-1/2 volume headers & data — used by `@brainana/imaging-math` (ROI warp, volume→surface projection) and the viewer. |
| **[ssh2](https://github.com/mscdex/ssh2)** | `^1.16.0` | 1.17.0 | MIT | Pure-JS SSH/SFTP client — backs the SFTP `DataSource` in `@brainana/core-server` for reading remote pipeline output. |

**Notes**

- `@niivue/niivue`, `fflate`, and `nifti-reader-js` are **not shipped as
  `node_modules`** in the desktop build — Vite bundles them into `dist/`
  (per `electron-builder.yml`).
- `ssh2` **is** shipped as a runtime `node_modules` dependency (declared in the
  root `package.json` `dependencies`). It has an optional native binary
  (`cpu-features`, a `.node` file) that `electron-builder` unpacks from the asar;
  `ssh2` falls back to pure JS if the ABI mismatches, so SFTP works regardless.

## 3. Development / build / packaging tools

Declared in the root `package.json` `devDependencies`. These build, typecheck,
and package the product but are not shipped to end users.

| Package | Declared range | Installed | License | Role |
| --- | --- | --- | --- | --- |
| **[typescript](https://www.typescriptlang.org/)** | `^5.7.0` | 5.9.3 | Apache-2.0 | Typechecking (`tsc --noEmit`) and the type-stripping the runtime relies on. |
| **[vite](https://vitejs.dev/)** | `^6.0.0` | 6.4.3 | MIT | Frontend dev server + production bundler for the viewer SPA. |
| **[electron](https://www.electronjs.org/)** | `^33.0.0` | 33.4.11 | MIT | Chromium shell for the desktop app — provides the WebGL2-reliable runtime NiiVue needs. |
| **[electron-builder](https://www.electron.build/)** | `^25.0.0` | 25.1.8 | MIT | Packages per-OS installers (AppImage/deb, dmg/zip, nsis). |
| **[@niivue/niivue](https://github.com/niivue/niivue)** | `^0.69.0` | 0.69.0 | BSD-2-Clause | Also listed here so it is available at build time for Vite bundling. |
| **[fflate](https://github.com/101arrowz/fflate)** | `^0.8.2` | 0.8.3 | MIT | (build-time, bundled) — see §2. |
| **[nifti-reader-js](https://github.com/rii-mango/NIFTI-Reader-JS)** | `^0.8.0` | 0.8.0 | MIT | (build-time, bundled) — see §2. |

There is **no dedicated test framework** — tests are plain `*_test.mjs` files
run by a custom runner (`scripts/run-tests.mjs`) using the Node built-in
`node:assert/strict`. No linter/formatter (ESLint/Prettier) is declared.

## 4. Internal workspace packages (first-party)

These are **not third-party** — they are the repo's own shared-platform layer
under `packages/*`, consumed by `apps/viewer`. Listed for completeness because
they appear as `@brainana/*` dependencies in the manifests.

| Package | Third-party deps | Purpose |
| --- | --- | --- |
| `@brainana/core-client` | fflate | Framework-agnostic browser platform (runtime client, source/filesystem client, session persistence, capability gate). |
| `@brainana/core-server` | ssh2 | Local HTTP runtime: loopback bind + token guard, DataSource registry (local + SFTP), cache, atomic export. |
| `@brainana/core-launcher` | *(core-server)* | Cross-platform launcher: mint token, scan port, start server, open browser. |
| `@brainana/core-desktop` | *(core-launcher, core-server)* | Electron shell that boots the loopback server in a hardened `BrowserWindow`. |
| `@brainana/imaging-math` | nifti-reader-js | Pure imaging math (ROI warp, volume→surface projection). Headless-testable. |
| `@brainana/niivue-kit` | *(peer: @niivue/niivue)* | Generic NiiVue helpers (orientation gizmo, landmark/crosshair markers). |
| `@brainana/ui` | *(none)* | Framework-free UI: design-token theme + self-hosted fonts + generic components. |

## 5. Bundled / self-hosted assets

Shipped inside `@brainana/ui` (`packages/ui/fonts/`, wired up in `theme.css`).
These are **redistributed font files**, so their licenses matter for compliance.

| Asset | Files | License | Notes |
| --- | --- | --- | --- |
| **[Source Sans 3](https://github.com/adobe-fonts/source-sans)** | `source-sans-3.woff2` (weight 400–700) | SIL OFL 1.1 | Self-hosted UI sans-serif; no CDN/network fetch. |
| **[IBM Plex Mono](https://github.com/IBM/plex)** | `ibm-plex-mono-400.woff2`, `ibm-plex-mono-600.woff2` | SIL OFL 1.1 | Self-hosted monospace for numeric/technical labels. |

> ✅ **Compliance:** the SIL Open Font License requires the license text to
> accompany redistributed font files. This is satisfied by
> [`packages/ui/fonts/OFL.txt`](../packages/ui/fonts/OFL.txt) (full OFL 1.1 text +
> both font copyright notices) and [`packages/ui/fonts/README.md`](../packages/ui/fonts/README.md)
> (per-file notice: family, weights, copyright, reserved font name, upstream).

## 8. Runtime & environment requirements

| Requirement | Value | Source |
| --- | --- | --- |
| Node.js | **≥ 22.18** | `engines.node` — needed for `import`-time type stripping the tests rely on. |
| Module system | ESM (`"type": "module"`) throughout | all `package.json` |
| TS target | `es2023`, `moduleResolution: bundler` | `tsconfig.base.json` |
| Browser baseline | last 2 majors of Chrome, Edge, Firefox, Safari | root `browserslist`; **WebGL2 required** (gated at runtime). |
| CI | GitHub Actions on ubuntu / macOS / windows-latest, Node 22 | `.github/workflows/ci.yml` |
| CI actions | `actions/checkout@v4`, `actions/setup-node@v4` | `.github/workflows/ci.yml` |
